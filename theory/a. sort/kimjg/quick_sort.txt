# 설명 (알고리즘)
 1. 배열내 임의로 pivot을 정한다
 2. pivot 보다 작은것을 앞으로 보낸다.
 3. 모든 배열을 비교했으면, 작은 수들의 다음칸으로 pivot을 옮긴다.
 4. 변경된 pivot의 위치를 기준으로 왼쪽, 오른쪽 배열에 1. 2. 3.번을 시행한다

# point
 - pivot을 정하고 비교하면 대부분의 OS 아키텍처에서 cache hit 비율이 높아져 성능이 높음.

# 안정성
 - False

# 정렬기법
 - 내부정렬

# 복잡도

1. 시간복잡도 :
    - best : O(nlog2(n))
    - avg : O(nlogx(n)) -> x 는 데이터 분포에 따라 결정
    - worst: O(n^2)

2. 공간복잡도 : 0
